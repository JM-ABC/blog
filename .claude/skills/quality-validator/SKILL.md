quality-validator 스킬


역할
  생성된 콘텐츠가 플랫폼 규격과 브랜드 스타일을 충족하는지 3단계로 검증하는 스킬

트리거 조건
  content-generator 스킬이 콘텐츠를 생성한 직후 자동 실행

입력
  content (필수): 검증할 콘텐츠 텍스트
  platform (필수): 대상 플랫폼 키
  brand_guide (필수): 브랜드 스타일 가이드 텍스트

출력
  검증 결과 딕셔너리
    passed: 통과 여부 (True/False)
    issues: 발견된 문제 목록
    suggestions: 수정 제안 텍스트 (실패 시)


검증 3단계

  1단계  스키마 검증 (스크립트)
    플랫폼별 필수 섹션이 존재하는지 체크
    instagram: "슬라이드" 키워드 7개 이상, "캡션" 섹션, "해시태그" 섹션
    reels: "후킹" 또는 "0-3초", "화면:", "자막:", 캡션 존재
    linkedin: 본문 길이 800자 이상, 해시태그 존재
    blog: 제목 존재, 본문 1500자 이상, 소제목(■) 3개 이상, TL;DR 요약 섹션(※ 이 글 핵심 요약) 존재
    threads: 본문 존재, 500자 이내 (단일 포스트 기준)

  2단계  규칙 기반 검증 (스크립트)
    금지 표현 검출
      정규식으로 문장 끝의 ~다, ~함, ~임 체 검출
      단, 인용문("...다") 안의 경우는 제외
    글자 수 범위 체크
      각 플랫폼별 최소/최대 글자 수 기준
    이모지 과다 사용 체크
      전체 대비 이모지 비율이 5% 이상이면 경고

  3단계  LLM 자기 검증 (에이전트 판단)
    브랜드 톤 적합성
      "친절한 사수" 느낌이 나는가
      구어체가 자연스러운가
    실무 인사이트 포함 여부
      단순 팩트 나열이 아닌 "왜"를 설명하는 문장이 있는가
      "실무에서 보면", "현장에서", "셀러 입장에서" 등의 관점이 포함되어 있는가
    전체 품질 점수
      1-10점 스케일로 평가, 7점 미만이면 실패


실패 시 처리
  자동 재시도: 검증 실패 피드백을 포함하여 content-generator에 재생성 요청
  최대 재시도: 2회
  2회 초과 시: 스킵 + 로그 (어떤 검증에서 실패했는지 기록)
