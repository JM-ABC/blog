content-generator 스킬


역할
  주제와 참고자료를 받아 특정 플랫폼에 최적화된 콘텐츠를 생성하는 스킬

트리거 조건
  사용자가 contentpipe generate 명령을 실행하거나,
  "콘텐츠 생성", "블로그 작성", "카드뉴스 만들어" 등의 요청이 들어올 때

입력
  topic (필수): 콘텐츠 주제 텍스트
  reference (필수): 참고자료 텍스트
  platform (필수): 대상 플랫폼 키 (instagram, reels, linkedin, blog, threads)
  brand_context (선택): 브랜드 가이드라인 텍스트

출력
  해당 플랫폼에 맞는 형식의 콘텐츠 마크다운 텍스트

처리 흐름
  1. prompts/base_system.py에서 공통 시스템 프롬프트 로드
  2. prompts/{platform}.py에서 플랫폼별 시스템 프롬프트 로드
  3. 브랜드 가이드라인이 있으면 시스템 프롬프트에 주입
  4. 주제 + 참고자료로 사용자 프롬프트 조합
  5. Claude API 호출
  6. 응답 텍스트 반환

참조 파일
  src/content_pipeline/platforms/ 하위 모든 생성기
  src/content_pipeline/prompts/ 하위 모든 프롬프트
  src/content_pipeline/client.py

플랫폼별 규칙
  instagram: 슬라이드 7-10장, 슬라이드당 50자, 디자인 노트, 캡션, 해시태그 10-15개
  reels: 30-60초, 후킹/도입/본문/마무리 구조, 화면/자막/BGM/효과 지시, 구어체
  linkedin: 1,300자 내외, 후킹 첫줄, 테이크어웨이, CTA, 해시태그 3-5개
  blog: 4,000-6,000자, 제목/TL;DR/본문/결론/출처/SEO태그 구조, SEO + GEO 최적화
         출처: 웹서치 기반 작성 시 참고한 기사·보고서 목록을 결론 다음에 명시 (매체명 + 연도)

         GEO 최적화 규칙 (AI 검색 인용 최적화, 구글 AI Overview / Perplexity 등)
           TL;DR 요약 섹션 필수: 제목 바로 아래 배치
             형식: ※ 이 글 핵심 요약 + 줄바꿈 후 · 로 핵심 수치·사실 3-4개 나열
             목적: AI가 답변 생성 시 이 블록을 인용할 확률을 높임
           핵심 용어 정의: 별도 섹션으로 분리하지 않고, 첫 등장 문장 뒤에 자연스럽게 한 문장으로 녹여넣기
             예시: "퀵커머스 시장은 ... 성장했어요. 퀵커머스는 주문 후 30분~1시간 내 배달을 목표로 하는 즉시배송을 말해요."
           수치 + 출처 세트: 모든 수치에 출처 명시 (AI 인용 신뢰도 향상)
           업체별·항목별 구조화 비교: AI Overview가 비교 요약 생성하기 좋은 포맷

         네이버 SEO 규칙
           제목에 연도, "총정리", "현황", "업체별", "비교" 등 C-rank 친화적 키워드 포함
           검색태그 30개: 핵심 키워드 + 세부 키워드 조합, 파일 맨 끝에 한 줄로 작성
           에디터 업로드 안내 (결과물 하단에 주석으로 포함):
             ■ 소제목 → 네이버 에디터에서 소제목(H3) 태그로 실제 적용 필요
             대표 이미지 1장 + 섹션별 이미지 2-3장 추가 권장
  threads: 500자 이내 단일 + 3-5개 스레드 버전, 캐주얼 톤
